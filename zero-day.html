Exploit Theory: The Zero-Day Lifecycle

Memory Corruption

A Zero-Day often targets software vulnerabilities that the developer is unaware of. By sending more data than a buffer can hold (Overflow), we can overwrite adjacent memory, including the **Return Address**.

Control Flow Hijacking

Once we overwrite the Return Address, we can point the CPU to execute our own code (Shellcode) hidden in the buffer. This bypasses the intended logic of the application.
